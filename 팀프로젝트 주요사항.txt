********** 해야할 것 **********
1. 컨텐츠 선정
 - 6주안에 할 수 있는것 기준(프로젝트 범위 설정)
 - 자료조사 필요(인터넷 or 현장답사)

2. 목적
 - 웹페이지의 디테일한 이유
 
3. 요구사항
 - 사용자 페이지 요구사항(타겟), 관리자 페이지 요구사항(사용하는 업체)
 - 목표와 비교하여 타당한 요구사항인지 확인
 - 결과물과 요구사항이 같아야 함.

4. 객체 관계도 작성
 - 요구사항에 맞춰 테이블 작성
 - 모델링 작업

5. DB 모델링
 - 객체 관계도를 참고하여 작성

6. DB 명세서
 - DB 모델링을 참고하여 작성

7. UseCase
 - 요구사항을 참고하여 작성

8. Site Map
 - 사용자용, 관리자용 작성

9. Site Layout
 - 사용자용, 관리자용 작성

10. 스토리 보드
 - 사용자용, 관리자용 작성
 - 각 페이지별 용도를 설명

11. 구조 명세서
 - workspace 디테일한 폴더 작성

12. 일정 관리
 - 전체, 개인별 일정 수립

13. 체크리스트
 - 각자 맡은 페이지에 관하여 기능 확인
